SÓLO UNO DE LOS MIEMBROS DEL GRUPO deberá entregar un único fichero .ZIP, con la siguiente información:

  1. Descripción del problema abordado. En pdf, en la memoria.

  2. Explicación de cómo se ha abordado el problema (descripción de qué se ha hecho en cada una de las secciones del fichero LEX/FLEX, para qué sirve cada línea del código LEX/FLEX, etc.). También en pdf, en la memoria

  3. Captura de pantalla describiendo un ejemplo de ejecución. En la memoria.

  4. Ficheros creados en la práctica.

  5. Ficheros .txt de ejemplo de entrada y/o cualquier fichero que sea necesario para probar el programa.

  6. Makefiles que se crean convenientes, para compilar el programa y también para ejecutar un ejemplo.
